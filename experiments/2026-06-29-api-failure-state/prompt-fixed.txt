# AI Task Packet v0.6: API Failure State Contract

## Scope
- Target paths: `experiments/2026-06-29-api-failure-state/fixed-app/**`
- Forbidden paths: `vibe-app/**`, repo root config, package manager files
- Dependencies policy: 依存追加なし。vanilla HTML/CSS/JavaScriptのみ。

## Product intent
日本語UIの「読書ログ同期ビューア」を作る。APIから読書ログを取得する体験を静的プロトタイプで再現し、利用者が成功時だけでなく、オフライン、タイムアウト、サーバーエラーから復帰できることを確認できるようにする。

## Functional requirements
- 読書ログ一覧を表示する。項目はタイトル、著者、読了日、メモ、同期状態。
- 検索入力でタイトル・著者・メモを絞り込む。
- API境界は `requestReadingLogs` または同等の関数としてUI描画から分離する。
- 画面上に状態切替用のデモ制御を置く。`data-scenario` または `scenario` で success / offline / timeout / server-error を確認できること。
- 再試行ボタンを置き、現在選ばれているscenarioで再読み込みできること。

## State design
- loading: 「読み込み中」をaria-live領域に出し、リストは更新中であることを表示する。
- empty: API成功かつ0件、または検索結果0件では「まだ表示できる読書ログがありません」等の空状態を表示する。
- success: 件数と最終同期時刻を表示し、ログ一覧を安全なDOM APIで描画する。
- offline: 「オフラインのため同期できません」と表示し、再試行ボタンを残す。
- timeout: AbortControllerまたは明示的なタイムアウト制御で「タイムアウトしました」と表示する。
- server_error: 500相当の「サーバーエラー」を表示する。
- recovery: エラー時も検索入力と既存データは保持する。利用者が再入力しなくてよい状態にする。

## Accessibility Contract
- `lang="ja"` と viewport を入れる。
- 状態変化は `aria-live="polite"` で伝える。
- 主要ボタンはbutton要素にし、44px以上のタップ領域を確保する。
- エラー/空/成功状態は見出しまたは明確なテキストで区別する。

## API Failure Contract
- API境界関数で success / offline / timeout / server-error を同じ戻り値または例外契約で扱う。
- timeout は `AbortController` を使うか、明示的な `setTimeout` と `timeout` 判定を持つ。
- エラー文言は日本語で、利用者が次に押すべき「再試行」が分かる。
- `console.log` で検索語や読書メモを出さない。
- 外部ネットワーク資産は使わない。
