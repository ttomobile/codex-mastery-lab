# Learning Log: AIDD-Spec v0.1 Validation 001

## うまく機能したこと

- AI Task Packetに対象ファイル、禁止事項、状態契約、アクセシビリティ契約が明記されていたため、実装範囲を絞れた。
- 外部通信禁止とビルドなしが明確だったため、依存なしの静的HTML/CSS/JSにできた。
- 静的契約チェッカーの対象が具体的だったため、検証コマンドを単純に保てた。

## 失敗または迷いやすい点

- media failureは空配列だったが、ユーザー制約では状態実装を求めているため、外部メディアを使わない説明としてUIに残した。
- 実ブラウザでのTab到達性は静的検査だけでは完全には保証できない。

## 次回Specへ戻す改善点

- 静的アプリ向けに「DOM実行テストを必須にするか、静的契約検査でよいか」を明記するとよい。
- failure_contractで空配列の場合、UIに説明を出すべきか、対象外としてよいかの判定規則があるとよい。
- loadingの表示時間やtimeout文言の最低条件をテンプレートに追加すると比較しやすい。
