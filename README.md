# Codex Mastery Lab

目的: M4 Mac mini (16GB RAM / 256GB SSD) という現実的な制約環境で、Codex CLI を使った安定的なシステム開発ワークフローを継続検証し、毎日「有料級の技術記事ドラフト」を生成する。

## 運用方針

- 無理に巨大プロジェクトを動かさない。小さな再現可能実験を積み上げる。
- Codex に任せる前に、仮説・評価指標・失敗条件を明文化する。
- すべての実験は git repo 内で行い、差分・ログ・テスト結果を保存する。
- 記事は「CTO視点」: テクニック紹介ではなく、再現性・リスク・運用設計まで書く。
- note への実投稿は認証/セッションが必要なため、初期運用は Slack への記事ドラフト納品。投稿権限を設定後に自動/半自動投稿へ移行。

## ディレクトリ

- `experiments/`: 日次実験ごとの作業場所
- `articles/`: note投稿用のMarkdownドラフト
- `assets/`: 図解、スクリーンショット、表
- `logs/`: Codex実行ログ、環境情報、検証結果
- `templates/`: 記事・実験計画テンプレート
- `backlog.md`: 検証テーマのバックログ

## 品質基準

1. 仮説がある
2. 実験手順が再現可能
3. Codexプロンプトと実行ログを残す
4. 成功/失敗を測る指標がある
5. スクリーンショットまたは図解を最低1点含める
6. 読者が明日から使えるチェックリストで締める
